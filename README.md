### **关于 [附件1：AtomGit代码仓使用指南](./资料/附件1：AtomGit代码仓使用指南.pdf) 的说明**

**代码仓权限申请**：
- 注册一个Atomgit账号，实名
- 填写 [附件2：参赛团队申请表](./资料/附件2：参赛团队申请表.xlsx) + 挑战杯报名官网盖章的报名表 + 团队人员签字的 [附件3：承诺函](./资料/附件3：承诺函.pdf) + 硬件邮寄地址，邮件主题为：代码仓权限及硬件申请-申报人所在单位-申报人姓名-作品名称-联系电话（例如：代码仓权限及硬件申请-XX 大学-张 XX-XX 方案-手机号），发送至邮箱 `dianhong_jbgs@163.com`
- 审核通过后在邮箱中找到邀请链接，提交申请，成为代码仓的“开发者”

**代码仓使用规范**
- 打开赛事官方代码仓，Fork到自己账号下。
- 在项目设置中，填写项目介绍，主分支选择main
- 根据电鸿SDK指南构建
- 填写README，包含功能简介、开发思 路、实现意义等说明
- 向官方代码仓提交PR，选择main分支（更健康的做法应该是提交基于main创建的branch，问过了，是可以的）

**开发时要注意的**
- 请不要在main分支上提交commit新内容
- 创建分支branch，在分支上开发新内容和提交
- 更加健康的习惯是在branch上再创建一个分支，并在新分支上开发和提交。该新分支通常在本地进行频繁的提交commit，完成某一模块的开发后可以合并到branch。**千万不要合并到main上**。
