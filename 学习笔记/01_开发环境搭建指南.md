# 01 开发环境搭建指南

> **目标：** 在一周内把开发需要用到的所有工具安装好，让电脑能编译 OpenHarmony 代码、能跑 Android 应用。
> **时间：** 第1周（5.17 - 5.24）
> **最终检验标准：** 在 Ubuntu 中能成功执行 `repo sync` 拉取 OpenHarmony 源码（至少走到一半不报错）；Android Studio 能创建项目并编译运行。

---

## 你需要安装什么？

整个项目涉及两个开发环境，需要两台"电脑"（其实是同一台电脑上跑两个环境）：

```
你的 Windows 电脑
  ├── WSL2 Ubuntu 虚拟机 ──→ 编译电鸿系统固件、编写摄像机端 C 代码
  └── Windows 原生环境   ──→ Android Studio 开发移动端 APP
```

> 💡 **知识卡片：为什么用 WSL2 而不是双系统？**
> WSL2（Windows Subsystem for Linux 2）是 Windows 内置的 Linux 运行环境。你不需要重启切换系统，Windows 和 Linux 文件互通（Windows 里可以直接访问 Linux 里的文件）。OpenHarmony 的编译工具链只支持 Linux，所以必须有一个 Linux 环境。

## 第一步：安装 WSL2 + Ubuntu 22.04

> **⏱ 预计耗时：** 30 分钟

### 1.1 打开 WSL 功能

打开 **PowerShell（管理员身份）**（右键点击开始菜单 → Windows PowerShell (管理员)），输入：

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

然后继续输入：

```powershell
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

> 💡 **知识卡片：WSL1 和 WSL2 有什么不同？**
> WSL1 是把 Linux 系统调用翻译成 Windows 系统调用来工作的（兼容层）。WSL2 是真正的轻量级虚拟机，运行完整的 Linux 内核。OpenHarmony 编译需要完整的 Linux 内核能力，所以必须用 WSL2。

**重启电脑。**

### 1.2 安装 Ubuntu 22.04

重启后，打开 **Microsoft Store**，搜索 "Ubuntu"，选择 **Ubuntu 22.04 LTS** 安装。

> 不要选 Ubuntu 24.04 —— OpenHarmony 官方推荐 20.04 或 22.04，24.04 可能出现兼容问题。

### 1.3 设置 WSL 默认版本为 WSL2

打开 **PowerShell**（不需要管理员），输入：

```powershell
wsl --set-version Ubuntu-22.04 2
```

> 如果这一步提示"WSL2 需要更新内核"，按提示去微软官网下载 WSL2 Linux 内核更新包安装即可。

### 1.4 设置 Ubuntu 用户名和密码

首次启动 Ubuntu 会提示创建用户名和密码，**记住这个密码**，后面 sudo 安装软件时会用到。

### 1.5 验证安装成功

在 Ubuntu 终端中输入：

```bash
uname -a
```

如果看到输出中包含 `microsoft` 字样，说明 WSL2 安装成功。例如：
```
Linux DESKTOP-XXX 5.15.153.1-microsoft-standard-WSL2 #1 SMP ... x86_64 x86_64 x86_64 GNU/Linux
```

> [!warning]
> windows启用WSL后，例如安装了Ubuntu系统，默认安装在C盘，导致我下面编译到一半的时候C盘满了。
> 迁移虚拟磁盘参考文档 [01-1-WSL虚拟磁盘迁移](01-1-WSL虚拟磁盘迁移.md)

## 第二步：在 Ubuntu 中安装开发必备工具

> **⏱ 预计耗时：** 20 分钟

### 2.1 更换为国内软件源（加速下载）

在 Ubuntu 终端中执行：

```bash
# 备份原来的源列表
sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup

# 编辑源列表
sudo sed -i 's/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g' /etc/apt/sources.list
sudo sed -i 's/security.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g' /etc/apt/sources.list

# 更新软件包列表
sudo apt update
```

> 如果你在学校的教育网环境下，清华镜像（tuna）速度最快；如果在家用电信/联通，也可以使用阿里云镜像 `mirrors.aliyun.com`。

### 2.2 安装基础开发工具

```bash
sudo apt install -y git curl wget vim openssh-client binutils build-essential \
    bison flex gcc-arm-linux-gnueabi gcc-arm-linux-gnueabihf gcc-aarch64-linux-gnu \
    python3 python3-pip python3-venv
```

逐条理解：
- `git`：版本管理工具，用来下载 OpenHarmony 源码
- `curl wget`：下载文件用的
- `vim`：终端文本编辑器
- `build-essential`：编译必备工具包（gcc, g++等）
- `bison flex`：语法分析器，编译 OpenHarmony 内核时需要
- `gcc-arm-*`：ARM 交叉编译器，编译摄像机端代码时需要用 ARM 指令集而不是 PC 的 x86 指令集
- `python3`：OpenHarmony 的构建脚本用 Python 编写

> 💡 **知识卡片：什么是交叉编译？**
> 你的电脑是 x86 架构的 CPU，但摄像机开发板用的是 ARM 架构的 CPU。x86 的程序不能直接在 ARM 上运行。交叉编译就是用 x86 电脑上的编译器，编译出能在 ARM 上运行的程序。`gcc-arm-linux-gnueabi` 就是这样一个"交叉编译器"。

### 2.3 配置 Git

```bash
git config --global user.name "你的名字"     # 比如 git config --global user.name "LiCo"
git config --global user.email "你的邮箱@example.com"
git config --global credential.helper store   # 记住密码，不用每次都输入
```

### 2.4 安装 repo 工具

OpenHarmony 使用 Google 的 `repo` 工具来管理多仓库源码（它本质上是对 git 的封装，可以一次性拉取上百个 git 仓库）。

```bash
# 创建 bin 目录并加入 PATH
mkdir -p ~/.local/bin
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 下载 repo 工具
curl -s https://storage.googleapis.com/git-repo-downloads/repo > ~/.local/bin/repo
chmod +x ~/.local/bin/repo

# 配置 repo 使用清华镜像（国内加速）
echo 'export REPO_URL="https://mirrors.tuna.tsinghua.edu.cn/git/git-repo"' >> ~/.bashrc
source ~/.bashrc
```

## 第三步：尝试下载 OpenHarmony 源码（了解流程）

> **⏱ 预计耗时：** 1~3 小时（取决于网速）
>
> ⚠️ **注意：** 这一步目前"了解"即可。完整的 OpenHarmony 源码大约 10GB+，下载时间长。你现在要做的是**理解这个过程**，等拿到硬件后按官方给的 SDK 文档操作。

> 💡 **知识卡片：电鸿和 OpenHarmony 的关系**
> **OpenHarmony** 是开源鸿蒙系统，由开放原子开源基金会运营。
> **电鸿（Electric-Hong）** 是在 OpenHarmony 基础上针对电力行业定制的版本，增加了电力设备协议适配、专用安全模块等。
> 本赛题中，你会拿到的是"电鸿 SDK"——它基本和 OpenHarmony 一致，但多了一些电力专用组件。所以现在学 OpenHarmony 完全没问题，后面迁移到电鸿只需要换 SDK。

### 3.1 创建源码目录

```bash
mkdir -p ~/openharmony
cd ~/openharmony
```

### 3.2 初始化 repo（以 OpenHarmony 4.1 Release 为例）

```bash
repo init -u https://mirrors.tuna.tsinghua.edu.cn/git/openharmony/manifest -b OpenHarmony-4.1-Release --no-repo-verify
```

> 这个命令的作用：下载一个"清单文件"，告诉 repo 工具需要从哪些 git 仓库、哪个分支拉取代码。
> 参数解释：
> - `-u`：清单文件仓库地址（用清华镜像加速）
> - `-b`：指定分支版本（4.1 Release 是较新的稳定版）
> - `--no-repo-verify`：跳过 GPG 验证（国内网络环境容易失败，跳过不影响使用）

> [!warning]
> 这里的清华镜像站用不了，尝试使用gitee站直接下载。
> ```bash
> # 修改 REPO_URL 为 gitee 镜像（之前配的是 tuna，也可以不修改）
> export REPO_URL="https://gitee.com/oschina/repo"
> repo init -u https://gitee.com/openharmony/manifest -b OpenHarmony-4.1-Release --no-repo-verify
> ```

### 3.3 同步源码

```bash
repo sync -c --no-repo-verify -j4
```

参数解释：
- `-c`：只下载当前分支的代码（不下载所有历史记录，节省空间和时间）
- `-j4`：4 个任务并行下载（如果你的电脑性能好、网速快可以改成 `-j8`）

**如果中途中断了，重新执行 `repo sync -c --no-repo-verify -j4` 会断点续传。**

> 💡 **可能出现的问题**
> - **"Connection timed out"**：网络超时，国内拉取 GitHub 上的仓可能不稳定。可以尝试配置代理，或者换成码云 gitee 镜像：
>   ```bash
>   repo init -u https://gitee.com/openharmony/manifest -b OpenHarmony-4.1-Release --no-repo-verify
>   ```
> - **"过早的文件结束符"**：个别文件下载失败，重新 `repo sync` 即可。

> 你会发现，做这一整步就是为了学会两个命令：`repo init` 和 `repo sync`。后面拿到电鸿 SDK 后，流程完全一样，只是把仓库地址换成官方提供的电鸿代码仓地址。

## 第四步：验证编译环境

> **⏱ 预计耗时：** 30 分钟

源码下载完成后，尝试编译一个最小的 OpenHarmony 镜像（不需要完整编译，太慢），目的是验证编译工具链是否安装正确。

```bash
cd ~/openharmony

# 安装编译依赖（OpenHarmony 提供的脚本）
bash build/prebuilts_download.sh

# 尝试配置编译选项
./build.sh --product-name rk3568 --ccache
```

> ⚠️ **注意：** 如果你没有完整下载源码（10GB+太大了），可以只做前两步，确认 `build/prebuilts_download.sh` 能跑通即可。等拿到电鸿 SDK 后，按照官方文档编译。

## 第五步：安装 Android Studio

> **⏱ 预计耗时：** 40 分钟

这是直接在 **Windows 环境下** 安装，不是在 WSL2 里。

### 5.1 下载 Android Studio

打开浏览器访问：https://developer.android.google.cn/studio

下载 Windows 版本（约 1GB）。

> 如果你在学校，教育网可以直接访问国内镜像；如果下载慢，可以用 Android Studio 中文社区的镜像地址下载。

### 5.2 安装

运行下载的安装包，一路默认设置即可。

> 💡 **知识卡片：Android Studio 装了什么？**
> Android Studio = 代码编辑器（基于 IntelliJ IDEA）+ Android SDK（开发工具包）+ 模拟器 + Gradle（构建工具）
> 后面开发 APP 时，你主要在 Android Studio 中写 Kotlin/Java 代码、设计 UI 布局、编译运行。

### 5.3 配置 SDK（首次启动时）

首次启动 Android Studio 会进入 Setup Wizard：
1. 选择 **Standard** 安装方式
2. SDK 版本建议勾选 **Android 12 (API 31)** 或 **Android 13 (API 33)**（覆盖绝大多数设备）
3. 等待 SDK 下载完成

### 5.4 创建 Hello World 验证环境

1. 点击 **New Project**
2. 选择 **Empty Views Activity**
3. 项目名：`CameraOAM`（OAM = O&M，运维的缩写）
4. Package name：`com.camera.oam`
5. Language：**Kotlin**
6. Minimum SDK：**Android 8.0 (API 26)**（覆盖 >95% 的设备）

创建完成后，点击右上角的绿色三角形 ▶ 运行。如果你有 Android 真机，用 USB 连接电脑运行；如果没有，创建模拟器运行。

> **检验标准：** APP 能在模拟器或真机上显示 "Hello World!" 画面，说明 Android 开发环境搭建成功。

## 第六步：安装常用工具（推荐）

### VS Code

在 Windows 中安装 VS Code（https://code.visualstudio.com/），安装后配置 WSL 远程开发：

1. 安装插件 **Remote - WSL**（微软官方）
2. 在 VS Code 中按 `F1`，输入 `WSL: Connect to WSL`，选择连接到 Ubuntu
3. 这样你就可以在 Windows 的 VS Code 界面中编辑 Ubuntu 里的代码了，比用 vim 舒服很多

> 💡 **为什么用 VS Code + WSL 远程？**
> 你的 C 代码在 Ubuntu 文件系统中（路径类似 `\\wsl.localhost\Ubuntu-22.04\home\你的用户名\`），在 Windows 里不好直接编辑。通过 Remote - WSL 插件，VS Code 就像直接在 Ubuntu 里运行一样，终端、文件树、调试器都无缝连接。
