> **方式：** 阅读 + 理解，不需要写代码。

## 一、电鸿 = OpenHarmony + 电力增强

```
OpenHarmony（开源鸿蒙）
  └── 电鸿（Electric-Hong）
       ├── 电鸿 = OpenHarmony 主干 + 电力行业专用组件
       ├── 增加了：电力设备通信协议（IEC 61850、Modbus 等）
       ├── 增加了：电力安全增强模块
       └── 增加了：针对摄像机的 HAL（硬件抽象层）
```

**一句话：** OpenHarmony 是"通用手机系统"的话，电鸿就是"电力行业定制版系统"。

你的摄像机不能跑 Windows，不能跑普通 Linux，它跑的是 **电鸿 OS**。你写的 C 程序是在电鸿系统上运行的。

---

## 二、你必须理解的 3 个核心概念

### 概念 1：摄像机端程序 = Linux 用户态进程

> 💡 **这是最重要的一个认知，请仔细读：**

电鸿系统底层是 **Linux 内核**（5.10 LTS 版本）。所以：

- 你在摄像机端写的 `O&M Service Daemon`（运维服务守护进程）→ 本质上是一个 **Linux 用户态进程**
- 你用 C 语言写它 → 编译后是一个 ELF 可执行文件 → 放到摄像机里运行
- 你需要用到 **socket 通信**（通过 UART 和星闪模组交互）
- 你需要用到 **文件操作**（读写摄像头设备节点 `/dev/video0`）
- 你需要用到 **线程/锁**（多线程处理：视频采集线程、通信线程、指令处理线程）

> 所以你学过的 Linux C 编程知识，除了"改内核驱动"那部分，**大部分都能用上**。

### 概念 2：OpenHarmony 的"应用"有两种

```
OpenHarmony 应用
  ├── Native 应用（C/C++）  → 你的 O&M Service 属于这类
  │   └── 编译产物：.so 动态库 或 ELF 可执行文件
  └── HAP 应用（ArkTS/JS） → 类似 Android 的 APK
      └── 编译产物：.hap 安装包（用在带屏幕的鸿蒙设备上）
```

**对你来说：** 摄像机本身没有屏幕（或者屏幕很小），所以不需要写 UI 应用。你的代码是 C/C++ 后台服务，通过星闪和手机 APP 通信。

### 概念 3：HDF（硬件驱动框架）

电鸿系统提供了一套统一驱动框架叫 **HDF（Hardware Driver Foundation）**。

```
你的 C 程序（O&M Service）
  ↓  调用 HDF 接口
HDF 框架
  ↓  调用具体驱动
摄像头驱动 / 云台驱动 / 星闪驱动
  ↓
硬件设备
```

有了 HDF，你不用直接操作寄存器或读设备手册——而是调用统一的 API：

```c
// 用 HDF 打开摄像头设备（伪代码，不是真实 API）
int fd = HdfDeviceOpen("camera", 0);
ioctl(fd, HDF_CAMERA_START_STREAM, &streamConfig);
```

> **现在你不需要写这段代码**，先理解有这个层次就行。拿到 SDK 后，我会帮你调具体 API。

## 三、摄像机端的"主菜单"——你的程序结构

这是你后面会写的 C 程序的大致结构，先有个印象：

```
O&M Service Daemon（运维服务守护进程）
│
├── main()
│   ├── 初始化日志系统
│   ├── 初始化 HDF 设备（摄像头、云台）
│   ├── 初始化星闪通信（打开 UART 端口）
│   ├── 启动"管理服务线程"
│   ├── 启动"视频推流线程"
│   └── 进入主循环（处理心跳、异常恢复）
│
├── 管理服务线程
│   ├── 监听星闪链路 → 收到指令包
│   ├── 解析指令 → 调用对应处理函数
│   ├── 执行操作（控制云台 / 查询参数 / 重启）
│   └── 发送响应包
│
├── 视频推流线程
│   ├── 从摄像头采集一帧（HDF 接口）
│   ├── 编码为 H.264（硬件编码器）
│   ├── 切成小数据包
│   ├── 通过星闪发送
│   └── 循环（15fps = 每 66ms 一帧）
│
└── 心跳/保活模块
    ├── 定时发送心跳包
    └── 检测连接断开 → 尝试重连
```

**3 个线程，各司其职。** 这就是你最终要在摄像机上跑起来的程序。

## 四、编译过程：你的 C 代码 → 摄像机上的程序

```
你在 Ubuntu 里写 C 代码
  ↓
使用交叉编译器（arm-linux-gnueabi-gcc）编译
  ↓
生成 ARM 架构的可执行文件
  ↓
通过烧录工具/ADB push 到摄像机开发板
  ↓
在电鸿系统上运行
```

> 💡 **你现在不需要操作这一步**，等你有了 SDK 和开发板，我会带你做。现在你只需要理解"我的代码不在电脑上直接跑，而是在 ARM 架构的开发板上跑"这件事。

## 五、需要记住的几个关键路径（拿到开发板后会用到）

```
摄像机开发板上：
  /dev/video0        → 摄像头设备节点（读取视频数据的入口）
  /dev/ttyS0         → 串口设备（和星闪模组通信的入口，也可能是 /dev/ttyUSB0）
  /data/oam_service  → 你的 O&M Service 程序路径
  /data/oam_config   → 配置文件路径
```
